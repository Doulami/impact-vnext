# payments-gpg\n\n**Purpose**: GPG Checkout payment provider plugin for Vendure.\n\n**Status**: Placeholder (M3 will implement)\n\n## Usage (API)\n\n```typescript\nimport { GPGPaymentPlugin } from '@impact/payments-gpg';\n\n// In Vendure config:\nplugins: [\n  new GPGPaymentPlugin({\n    endpoint: process.env.GPG_ENDPOINT,\n    merchantId: process.env.GPG_MERCHANT_ID,\n    secret: process.env.GPG_SECRET,\n    webhookSecret: process.env.GPG_WEBHOOK_SECRET,\n  }),\n]\n```\n\n## Endpoints\n\n- `POST /create-checkout`: generate signed redirect URL\n- `POST /webhooks/gpg`: webhook handler (authorized, captured, failed, refunded)\n- `POST /replay-webhook`: idempotent webhook replay (admin)\n\n## Kill Switch\n\nGate with `payments.gpgEnabled` flag:\n- If false: payment method not registered; checkout falls back\n- No code changes needed; toggle in Strapi StoreConfig\n\n## See Also\n\n- docs/specs/payment-gpg.md – Full specification\n- docs/runbooks/ops.md – Payment reconciliation procedures\n"
