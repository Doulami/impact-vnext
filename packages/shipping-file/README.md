# shipping-file\n\n**Purpose**: Automated order export to file (CSV) for fulfillment via SFTP or HTTPS.\n\n**Status**: Placeholder (M4 will implement)\n\n## Usage (API)\n\n```typescript\nimport { ShippingFilePlugin } from '@impact/shipping-file';\n\n// In Vendure config:\nplugins: [\n  new ShippingFilePlugin({\n    schedule: process.env.SHIP_EXPORT_SCHEDULE, // cron\n    protocol: process.env.SHIP_REMOTE_PROTOCOL,\n    sftp: {\n      host: process.env.SHIP_REMOTE_HOST,\n      user: process.env.SHIP_REMOTE_USER,\n      password: process.env.SHIP_REMOTE_PASSWORD,\n    },\n  }),\n]\n```\n\n## Endpoints\n\n- Scheduled export: cron job (SHIP_EXPORT_SCHEDULE)\n- Manual export: `POST /admin/orders/export-shipping`\n- Re-export: `POST /admin/orders/export-shipping?batchId=...`\n\n## File Schema\n\nCSV with columns: order_id, customer_name, address_*, items_skus, items_qty, weight_kg, special_notes, cod_flag, carrier_service, etc.\n\nSee docs/specs/shipping-file-bridge.md for full spec.\n\n## Kill Switch\n\nGate with `shipping.fileBridgeEnabled` flag:\n- If false: no auto-export; manual export disabled\n\n## See Also\n\n- docs/specs/shipping-file-bridge.md – Full specification\n- docs/runbooks/ops.md – Manual export & re-export procedures\n"
